Build Sequence

MStep 1 — Config caps

Files: backend/src/config.ts, backend/src/test-units.ts.

Do: add orchFanoutConcurrency (env ORCH_FANOUT_CONCURRENCY, default 4) and orchMaxTargets (env ORCH_MAX_TARGETS, default 10).

Tests: defaults are 4 and 10. Gate: npm run test:units + npm run typecheck. (AC3, AC7)

MStep 2 — Aggregation (rankResults)

Files: create backend/src/orchestrator2/aggregate.ts; test-units.ts.

Do: pure rankResults(results: TaskResult[]): TaskResult[] — sort best-first: ok-and-not-lowConfidence first, then by descending pitch score (read from result.summary? no — rank by an explicit score field). Pass a numeric score on ToolResult (add optional score?: number to ToolResult in tools.ts, set from gate.score) so ranking is data-driven, not string-parsed.

Tests: ok+high-score ranks above ok+low-score; failed/low-confidence rank last; stable for empty. Gate: npm run test:units. (AC5)

MStep 3 — Engine bounded-parallel execution + ranking

Files: backend/src/orchestrator2/engine.ts; test-units.ts.

Do: replace the sequential pending-task for loop with a runPool(tasks, K, fn) helper that keeps at most K promises in flight; K = config.orchFanoutConcurrency (injectable via an engine option for tests). Call rankResults(results) before synthesizer. Preserve per-task gate + events + status.

Tests (LLM-free): with K=2 and 5 tasks, peak in-flight never exceeds 2 and all 5 run (instrument a counter in the mock tool); results passed to synthesizer are ranked. Gate: npm run test:units — existing engine tests still green. (AC3, AC4, AC5)

MStep 4 — Planner multi-target + expansion + cap

Files: backend/src/orchestrator2/planner.ts; test-units.ts.

Do: buildPlannerPrompt instructs explicit-list → one task each; descriptive → expand from own knowledge up to maxTargets (no web). planGoal(goal, hints, maxTargets) caps the returned tasks at maxTargets, dropping extras; worker passes config.orchMaxTargets.

Tests: prompt mentions the cap and "from your own knowledge / no web"; a post-parse cap helper trims a 15-item list to 10 (pure helper capTasks). Gate: npm run test:units + typecheck. (AC1, AC2, AC7)

MStep 5 — Coverage-aware goal grader

Files: backend/src/orchestrator2/goal-grader.ts; test-units.ts.

Do: buildGraderPrompt grades coverage — fraction of targets with a confident pitch — not a single perfect output; met when coverage is sufficient.

Tests: prompt includes coverage/"how many targets" language and the work summaries. Gate: npm run test:units. (AC6)

MStep 6 — ADR + final verification

Files: create docs/adr/0004-fanout-parallel-execution.md; CHANGELOG.md.

Do: record the parallel-in-engine vs fan_out-tool decision + caps. Run full suite + both typechecks.

Tests: whole test:units green; backend+frontend typecheck. Live e2e (multi-target) deferred — manual, needs services. Gate: green suite. (AC8)
