Problem

Increment 1 runs targets one task at a time, sequentially. A batch goal ("research these 8 fintechs" or "the top 5 EU payment processors") works only serially — slow (N full pipelines back-to-back) — and the orchestrator has no notion of covering a list and comparing the results. The Inc 1 e2e also showed the goal grader is too strict for batch intent: it demands a single perfect output and rejects a set of decent ones.

User Impact

A user can state a multi-target goal and get all prospects researched concurrently with a ranked, honest coverage report — not a slow serial crawl. Explicit lists run as given; descriptive goals are expanded by the planner from its own knowledge (no web — that is Increment 4).

Acceptance Criteria

  • OPENAC1 — A goal with an explicit list of T targets produces T run_research_pipeline tasks (one per target).
  • OPENAC2 — A descriptive goal ("top 5 X") makes the planner emit up to ORCH_MAX_TARGETS tasks from its own knowledge, with no web call; planner-expanded results are flagged as such in the output.
  • OPENAC3 — The engine runs independent pending tasks concurrently, with never more than ORCH_FANOUT_CONCURRENCY (default 4) in flight at once.
  • OPENAC4 — Each task is gated independently (reusing the Inc 1 result gate); one failing/low-confidence target does NOT abort the others.
  • OPENAC5 — An aggregation step ranks completed results by pitch score (with lowConfidence de-prioritized) before the synthesizer runs, so the final answer presents targets best-first.
  • OPENAC6 — For a multi-target goal, the goal grader grades coverage ("M of N targets have confident pitches") rather than demanding one perfect output; a fully-covered batch returns goalMet=true.
  • OPENAC7 — Target count is capped at ORCH_MAX_TARGETS (default 10); excess targets are dropped with a logged/emitted note — no silent truncation.
  • OPENAC8 — Engine bounded-parallel execution, the ranking function, and the coverage-grader prompt builder are covered by LLM-free unit tests (concurrency cap respected; ranking order; coverage prompt content).

Out of Scope

  • Web-based discovery of the target list (Increment 4).
  • Granular per-phase tools (Increment 3).
  • Cross-target dedup/merge beyond ranking; comparison narratives.
  • Resuming a partially-failed fan-out.

Open Questions

  • Resolved: list source = explicit + planner expansion (no web); execution = bounded parallel in the engine (no new tool); caps = K=4 concurrency, N=10 targets.
