Spec: BILTIQ-004 — Company-level chat (P2 of Projects)

Task BILTIQ-004Owner @harishCreated 2026-06-23Wall-clock MCompliance on_prem_preferredDepends on BILTIQ-003 (projects, grouping)

Problem

Chat today reasons over a single run (/runs/:id/chat). When a company has multiple research runs, there is no way to ask questions or refine pitches against the company's full accumulated history.

User Impact

From a project's company group, the user opens a chat that reasons over all of that company's runs (combined signals, profiles, pitches), so they can ask cross-run questions and improve outreach using everything known.

Acceptance Criteria

  • OPENAC1 — POST /api/projects/:id/companies/:company/chat {message} aggregates context from ALL runs in that project+company and returns an assistant reply via llm/client.
  • OPENAC2 — Company chat history persists keyed by (projectId, normalized company) and is returned by GET on the same path; replies append to it.
  • OPENAC3 — The aggregated context is built by a pure, unit-tested helper from a list of runs (dedupes/limits signals; includes each run's company/summary/top signals).
  • OPENAC4 — The project detail page exposes a "Chat" affordance per company that opens a drawer talking to the company chat endpoint; messages render and persist across reloads.
  • OPENAC5 — A company with zero runs cannot be chatted (button disabled / 400); existing per-run chat is unchanged.
  • OPENAC6 — The context-builder helper has LLM-free unit tests (multiple runs merged; empty list; signal cap).

Out of Scope

  • P3 (pipeline builds-on-prior), P4 (project synthesis). Cross-company/project-level chat. Streaming responses.

Open Questions

  • Resolved: company key = normalized lowercase company; history in a small CompanyChat collection keyed by (projectId, companyKey).
